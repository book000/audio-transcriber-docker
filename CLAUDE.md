# CLAUDE.md

## 目的
このドキュメントは、Claude Code の作業方針とプロジェクト固有ルールを示す。

## 判断記録のルール
1. 判断内容の要約を記載する
2. 検討した代替案を列挙する
3. 採用しなかった案とその理由を明記する
4. 前提条件・仮定・不確実性を明示する
5. 他エージェントによるレビュー可否を示す

## プロジェクト概要
- 目的: Web Speech API を使用して、動画や音声ファイルの音声を自動的に文字起こしする。
- 主な機能: Puppeteer を使用した Chrome の自動操作、Web Speech API による文字起こし、Docker による環境構築。

## 重要ルール
- 会話言語: 日本語
- コミット規約: Conventional Commits
- コメント言語: 日本語
- エラーメッセージ言語: 英語

## 環境のルール
- ブランチ命名: Conventional Branch (`feat/`, `fix/` などの短縮形を使用)
- GitHub リポジトリ調査: `gh` コマンドや `ls`, `grep` 等を使用して調査する
- Renovate PR: 既存の Renovate PR に対して追加コミットや更新を行わない

## コード改修時のルール
- 日本語と英数字の間には半角スペースを挿入する
- エラーメッセージに絵文字が含まれる場合は、全体で絵文字を統一する
- TypeScript の `skipLibCheck` は使用禁止（本プロジェクトは現状 JavaScript だが、導入時は遵守すること）
- 関数やインターフェースには JSDoc 等を日本語で記載することを推奨する（既存コードには記載されていないため、これは新規コードに対する推奨事項であり、既存コードへの遡及適用は必須ではない）

## 相談ルール
- Claude Code: 実装レビュー、局所設計、整合性確認
- Gemini CLI: 外部仕様、最新情報確認
- 指摘への対応: GitHub Copilot Code Review が出力する「信頼度」スコア（0〜100）を基準とし、スコア 50 以上の指摘には原則として対応すること
  - ただし、ビジネス要件や既知の設計意図と明確に矛盾する場合は、対応しない理由を「判断記録のルール」に従って記録したうえで例外としてよい

## 開発コマンド
### Docker 操作
```bash
# イメージのビルド
docker build -t audio-transcriber .

# 実行（スクリプト経由）
./start.sh
./start-mount.sh
```

## アーキテクチャと主要ファイル
- `main.js`: Puppeteer を使用したメインロジック。Chrome の起動、Web Speech API の制御、結果の保存を行う。
- `Dockerfile`: Node.js 17 ベースの環境。Chrome, PulseAudio, 日本語フォント等のインストールを行う。
- `entrypoint.sh`: コンテナ起動時に実行されるスクリプト。
- `start.sh` / `start-mount.sh`: ホスト側からコンテナを起動するためのヘルパースクリプト。

## 実装パターン
- Puppeteer を使用したブラウザ自動操作
- `page.exposeFunction` によるブラウザコンテキストと Node.js 間の通信
- `spawn("paplay", ...)` による音声再生と仮想マイクへの入力

## 作業チェックリスト

### 新規改修時
1. プロジェクトを理解する
2. 作業ブランチが適切であることを確認する
3. 最新のリモートブランチに基づいた新規ブランチであることを確認する
4. PR がクローズされた不要ブランチが削除済みであることを確認する
5. 必要に応じて依存関係をインストールする

### コミット・プッシュ前
1. Conventional Commits に従っていることを確認する
2. センシティブな情報が含まれていないことを確認する
3. Lint / Format エラーがないことを確認する
4. 動作確認を行う

### PR 作成前
1. PR 作成の依頼があることを確認する
2. センシティブな情報が含まれていないことを確認する
3. コンフリクトの恐れがないことを確認する

### PR 作成後
1. コンフリクトがないことを確認する
2. PR 本文が最新状態のみを網羅していることを確認する
3. `gh pr checks <PR ID> --watch` で CI を確認する
4. Copilot レビューに対応し、コメントに返信する
5. コードレビューを実施し、指摘対応を行う

## リポジトリ固有
- Chrome の Web Speech API の挙動に依存している。
- 音声の入出力に PulseAudio の仮想デバイスを利用しているため、環境設定が重要となる。
